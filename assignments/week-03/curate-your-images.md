## AI Generated Assets

AI is used only for these specific non-representational assets:

1. **Favicon concepts** — The monogram-based JC favicon was iterated on in an AI image generator to find a clean, recognizable 16×16 pixel silhouette. AI helped test variations of corner rounding, stroke weight, and spacing before the final version was exported as SVG.

2. **Abstract hero background** — The hero section background uses a soft abstract gradient/pattern generated by AI. This is decorative only — it has no informational content and does not misrepresent the application.

3. **Small decorative graphics** — Minor visual ornaments (e.g., subtle card accents, divider decorations) may be AI-generated. These are purely cosmetic and do not affect the user's understanding of the product.

Everything else — every page screenshot, every UI state, every responsive layout capture — is a real browser capture from the deployed or locally running application.

## Rejected Images

These AI-generated image concepts were tried and rejected:

### 1. "Developer at desk" hero illustration

A photorealistic AI illustration of a developer working at a dual-monitor setup.

**Rejected because:** Looked too generic — every portfolio site has this exact image. It added no information and made the page feel templated rather than authentic. Replaced with a real screenshot of the actual application layout.

### 2. "Floating UI elements" composite

An AI image showing floating phone screens, browser windows, and code snippets arranged as a collage.

**Rejected because:** Inconsistent with the portfolio's minimal, content-first design language. The collage style clashed with the clean card-based layout defined in the Identity Kit. It distracted from the real project screenshots that should be the focus.

### 3. "Rocket launch" metaphor graphic

An abstract AI illustration of a rocket lifting off, intended to represent "launching projects."

**Rejected because:** Unrealistic UI metaphor — the portfolio is about software engineering, not space travel. The image was conceptually disconnected from the actual work and would confuse visitors expecting to see real applications.

## Real Over AI Decisions

These five decisions prioritise real screenshots over AI-generated alternatives:

1. **Projects page** — Each project card could have had an AI-generated mockup thumbnail. Instead, the page uses no thumbnails at all — real screenshots will be added once the projects are more mature. This avoids fake-looking placeholder images.

2. **Settings form** — AI could have generated a fake filled-form screenshot. Instead, a real browser capture of the form with typed-in values is used. This shows actual React Hook Form validation states (error messages, disabled button).

3. **Health check** — AI could have mocked a fake API response screenshot. Instead, the screenshot shows a real JSONPlaceholder fetch with a live timestamp, proving server-side data fetching works.

4. **Navbar active state** — AI could have generated a mock navigation bar. Instead, a real screenshot of the Navbar rendered with `usePathname()` active highlighting is used.

5. **Responsive layouts** — AI could have generated mockups of how the site looks on mobile. Instead, real browser captures at 375px, 768px, and 1280px viewports are used.

## Prompt Used For AI Images

### Abstract Hero Background

```
Generate a soft abstract background image for a developer portfolio hero section.

Style:
- Minimal and modern
- Subtle gradient from indigo-600 to slate-900
- Gentle geometric shapes (circles, blurred orbs) in the background
- No text, no UI elements, no people
- Should not distract from foreground content
- Aspect ratio: 1440×600px
- Format: WebP or SVG

The background is purely decorative and sits behind a centered heading, subtitle, and two CTA buttons.
```

### Favicon Concept

```
Design a minimal monogram favicon for initials "JC".

Constraints:
- Square aspect ratio, recognizable at 16×16px and 32×32px
- Single-color silhouette that works on light and dark backgrounds
- Clean sans-serif letterforms (similar to Space Grotesk geometry)
- Slightly rounded corners for approachability
- No gradients, no fine details that blur at small sizes
- Style: modern, technical, developer-focused

This will be used as the site favicon and browser tab icon.
```
